About tail.DateTime

The tail.DateTime script is a fork of MrGuiseppes Pure JS Calendar script, which contains the following changes:

  • A Time Picker (Hours, Minutes, Seconds)
  • A Month-Based browse view
  • A slightly different date/time output format
  • A slightly different HTML class/id layout
  • A different JS structure / implementation / constructur (Prototyping)
  • Key Listener for 'Escape' (Close) and 'Enter' (Use Today / First Day of the month)
  • Auto-Close on click outside of the calendar popup / input element
  • Instance Caching + Calendar Caching with the first day in the week
  • Default value depending on the input field
  • "Translatable" Strings through the global variable
  • And many more... Check out the Changelog for details.

Install & Embed

You can download and install the tail.DateTime package using one of the 2 buttons (.tar or .zip) above in the header, or by callin NPM npm install tail.DateTime or Yarn yarn add tail.DateTime directly in your console, if you have the respective package manager installed of course.

You can also use the CDN jsDelivr or unpkg to receive the tail.DateTime script without to download the package itself: 

https://cdn.jsdelivr.net/npm/tail.datetime@latest/
https://unpkg.com/tail.datetime

No matter which way you take, you still need to embed the JavaScript as well as the Stylesheet in your HTML document and please don't forget to embed the desired language JavaScript and theme stylesheet file too. We recommend to load the stylesheets within the HTML header and the JavaScript files directly after the opened <body> HTML tag. 

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />
        <title>Example Site</title>

        <link type="text/css" rel="stylesheet" href="css/tail.datetime.css" />
        <link type="text/css" rel="stylesheet" href="css/tail.datetime.white.css" />
    </head>
    <body>
        <script type="text/javascript" src="js/tail.datetime.min.js"></script>
        <script type="text/javascript" src="langs/tail.datetime-de.js"></script>
    </body>
</html>

How to Use

The tail.DateTime script needs a input element, where it builds up the respective environment. Just add this HTML Element anywhere you need and call the tail.DateTime() function, optimally within the DOMContentLoaded event to prevent beforeLoading failures. The global tail.DateTime() function allows 2 arguments. The first MUST be

  • ... a (HTML) Element Object (getElementById, querySelector)
  • ... a (HTML) Element List Object (getElementsBy*, querySelectorAll)
  • ... an Array with Element Objects ([getElementById, getElementById])

or just a STRING selector like "input[type="datetime-local"]", which equals the querySelectorAll method.

The second parameter CAN be an object with respective key => value based options, check out the list below to find all available options. You can also skip this and just use the default settings. 

document.addEventListener("DOMContentLoaded", function(){
    tail.DateTime("input[type='datetime-local']");
});

How to Configure

The tail.DateTime DateTime Picker should now already work, but you can still configure the interface, the design as well as the behavior with the following options: 

document.addEventListener("DOMContentLoaded", function(){

    // The settings below are the defaults!
    tail.DateTime("input[type='datetime-local']", {
        static: null,
        position: "bottom",
        classNames: "",
        dateFormat: "YYYY-mm-dd",
        timeFormat: "HH:ii:ss",
        dateRanges: [],
        weekStart: "SUN",
        startOpen: false,
        stayOpen: false,
        zeroSeconds: false
    });
});

Available Options

static
string
Pass an valid selector of an element where the Calendar Popup should be append to, leave it on null to use the absolute position calculation.
position
string
Sets the position of the DateTime popup field to "top", "left", "right" or "bottom". This option does not work if you pass an valid selector on static!
classNames
string|array
Adds custom class names to the main DateTime container element, pass and array or an space-separated list!
dateFormat
string|bool
Pass an valid Date/Time Format (see below) or pass false to disable the Date picker.
timeFormat
string|bool
Pass an valid Date/Time Format (see below) or pass false to disable the Time picker.
dateRanges
array
This option allows you to create one or more selectable periods of time. Each period defines itself through an array, where the first value if the start-point and the last one the end-point (which is also optional). You can use a Date object as value, a string as YYYY-MM-DD or the (translated) name of a weekday (See tailDateTime.strings.shorts).

Example

The following Example limits the selectable area to:

  • From 2018-10-12 up to 2018-10-26
  • From 2018-11-04 up to 2018-11-30
  • From 2018-12-01 up to 2018-12-31
  • In all cases only from Monday up to Friday (Sunday and Saturday are not selectable)

The last array ["MON", "FRI"] limits only all selections before! If you want to just use a selection between Monday and Friday in general, add a third parameters with true. 

tail.DateTime("input[type='datetime-local']", {
    dateRanges: [
        [new Date(2018, 9, 12), new Date(2018, 9, 26)],
        ["2018-11-4", "2018-11-30"],
        [new Date(2018, 11, 1), new Date(2018, 11, 31)],
        ["MON", "FRI"]
    ]
});
weekStart
string
Pass the (translated) string of the global variable tailDateTime.strings.shorts to change the shown first day in the week. (If no translation has been made, you can choose between: "SUN", "MON", "TUE", "WED", "THU", "FRI" and "SAT")
startOpen
bool
Use true to show the DateTime Picker directly after the initialization.
stayOpen
bool
Use true to keep the DateTime Picker open, even if an selection has been made or any other closing event has been triggered! (You can still close the DateTime Picker with the close() method!)
zeroSeconds
bool
Use true to set the seconds to 00. This would be happen once, during the initialization.
